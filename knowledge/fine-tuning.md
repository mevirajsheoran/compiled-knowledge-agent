# Fine-Tuning

## Summary
Fine-tuning adapts a pre-trained language model to a specific domain or task by training it on a curated dataset, embedding domain knowledge and specific behaviors directly into its weights. It provides consistent style but is slower and more expensive to update than RAG.

## Key Points
- Best for consistent style, tone, or domain-specific behavior
- Requires high-quality training data
- More expensive and slower to iterate than RAG
- Changes are baked into the model weights
- Good for tasks with stable requirements
- Adapts an LLM to a specific domain or task via further training.
- Embeds consistent style, tone, or specific behaviors directly into model weights.
- Requires high-quality training data for optimal results.
