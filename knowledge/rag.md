# RAG

## Summary
Retrieval-Augmented Generation (RAG) combines external information retrieval with answer generation, making it highly effective for dynamic knowledge bases like those used by support bots. It grounds responses in real data, reducing hallucinations.

## Key Points
- Useful when knowledge changes frequently
- Often better than fine-tuning for dynamic documentation
- Quality depends heavily on retrieval quality
- Can reduce hallucinations by grounding responses in real data
- Works well with large, evolving knowledge bases
- Fetches relevant external documents at query time.
- Crucial for support bots with frequently updated knowledge bases.
- Reduces hallucinations by grounding responses in retrieved facts.
